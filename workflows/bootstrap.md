# Bootstrap Workflow

Use this for `mode: bootstrap`.

## Goal

Turn an early product idea or project brief into a usable Linear task map.

## Steps

1. Read the project brief from the Linear issue.
2. Identify goals, users, surfaces, deliverables, unknowns, and constraints.
3. Split the project into Linear-ready tasks.
4. Classify every task with one mode:
   - `mode: explore`
   - `mode: prototype`
   - `mode: build`
   - `mode: park`
5. Add acceptance criteria or expected output for each task.
6. Recommend the first execution order.
7. If Linear tools are available and the product owner asked for creation, create the issues.
   Otherwise, post the task map as a comment.

## Output

The result should include:

- Project goal.
- Proposed Linear issues.
- Mode for each issue.
- Deliverable for each issue.
- Dependencies.
- Recommended first 3 tasks.

## Merge Policy

No code merge. This workflow creates or refines project structure.
